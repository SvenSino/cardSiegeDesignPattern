\section{Einleitung}

\subsection{Motivation}

Entwurfsmuster sind bewährte Lösungsschablonen für wiederkehrende Probleme in der
objektorientierten Softwareentwicklung. Als gemeinsame Sprache für Architekturentscheidungen
haben sie sich in der Praxis fest etabliert -- ihr eigentlicher Wert zeigt sich
jedoch erst im Kontext eines realen Entwurfsproblems. Reine Theorie lässt den
Nutzen oft abstrakt wirken; erst wenn ein Muster ein konkretes Problem löst,
wird seine Wirkung greifbar.

Dieses Projekt setzt neun Entwurfsmuster in \textbf{CardSiege} um, einem
rundenbasierten Kartenspiel mit Tower-Defense-Mechaniken in Java und libGDX.
Spieleentwicklung ist ein besonders geeignetes Anwendungsgebiet, weil Spiele
komplexe, sich ständig ändernde Zustände verwalten, viele Objekte eng
miteinander interagieren und Erweiterbarkeit von Anfang an gefordert ist.
Dabei löst jedes Muster ein konkretes Architekturproblem, das sich ohne es
in Kopplung, Codeduplikation oder mangelnder Wartbarkeit niedergeschlagen hätte.

\subsection{Spielkonzept}

Der Spieler verteidigt eine Basis gegen zwölf Gegnerwellen, indem er auf einem
20$\times$20-Rasterfeld strategisch Türme platziert. Das Besondere liegt im
kartenbasierten Ressourcensystem: Statt Türme direkt zu kaufen, zieht der Spieler
zu Beginn jeder Runde drei Karten aus einem gemischten Deck. Diese ermöglichen
unterschiedliche Aktionen, Turmbau, Attribut-Buffs, Ressourcengewinn oder
Sondereffekte wie das Verlangsamen von Gegnern.

Jeder Spielzug folgt einem festen Viererzyklus: \textit{Draw} (Karten ziehen,
Energie auffüllen) $\rightarrow$ \textit{Play} (Karten ausspielen, Türme
positionieren) $\rightarrow$ \textit{Wave} (Gegnerwelle läuft, Türme greifen
automatisch an) $\rightarrow$ \textit{Resolve} (kurze Pause, nächster Zug
beginnt). Das Spiel endet mit Sieg nach zwölf abgewehrten Wellen oder mit
Niederlage, sobald die Basis auf null Lebenspunkte fällt.

\subsection{Technische Voraussetzungen und Projektstart}

CardSiege ist als Multi-Modul-Gradle-Projekt strukturiert. Das Modul
\texttt{core/} enthält die plattformunabhängige Spiellogik, \texttt{lwjgl3/}
stellt den Desktop-Launcher bereit. Voraussetzung ist ein installiertes
Java~21~JDK; eine separate Gradle-Installation ist nicht erforderlich, da der
mitgelieferte Gradle-Wrapper alle weiteren Abhängigkeiten automatisch
herunterlädt.

\begin{lstlisting}[language=bash,caption={Build und Start \"{u}ber den Gradle-Wrapper},label=lst:build]
# Projekt bauen
./gradlew build

# Spiel starten (Desktop)
./gradlew lwjgl3:run
\end{lstlisting}

Unter Windows wird statt \texttt{./gradlew} der Befehl \texttt{gradlew.bat}
verwendet. Der Einstiegspunkt ist die Klasse
\texttt{td.lwjgl3.Lwjgl3Launcher}; das Spielfenster öffnet sich mit
1200$\times$700~Pixeln.
